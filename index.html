What's is HTML

HTML is a hypertext markup language. It is used to place elements on a web page: text, images, tables, and video.

Before displaying a page, the browser reads the HTML document and passes each element in the stream, in the order in which it appears. CSS can change their location regardless of their ordinal location in the document and move them to any point in the viewing area.

As already mentioned, HTML consists of elements. There are two types of elements: single (not closing tag) and with a closing tag. Let's consider the option with a closing tag. In the example below, the element has an opening tag <p> and a closing tag </p>. Remember, you always need to close elements with a closing tag.

<p> opening tag and a closing tag </p>

As you may have noticed in the example below, single elements do not have closing tags. You may also notice that the element contains two attributes. The first <src> source is needed to display the image by specifying its full path (absolute) or relative. The second <alt>is an alternative image in the form of text in case the image was not loaded or there was a failure, as well as for the screen reader.

<img src="URL" alt="description image" >
Basic element HTML

Every HTML document starts with a basic structure of elements. Here is an example of a basic HTML document:

            
                    <!DOCTYPE html>
                    <html lang="en">
                    <head>
                        <meta charset="UTF-8">
                        <meta name="viewport" content="width=device-width, initial-scale=1.0">
                        <title>Document</title>
                    </head>
                    <body>
                            
                    </body>
                    </html>

The <!DOCTYPE html> declares the version of html, currently it is the fifth version. The <html lang="en"> element is the root element in a document that contains all other elements. It can also contain an attribute such as lang="language" to indicate the language of the page.

Attributes in HTML are additional properties that are added to tags (elements) to change their behavior, appearance, or provide additional information. Attributes are always specified inside the opening tag and consist of a name and value separated by an equal sign (=). The attribute value is usually enclosed in quotation marks (single ' ' or double " "). There are also attributes that have no value, for example, "disabled".

    List of Basic HTML Elements.

What's is CSS

CSS, or Cascading Style Sheets, is a stylesheet language used to describe the presentation of a document written in HTML or XML. It controls the layout, colors, fonts, spacing, and overall visual appearance of web pages. CSS separates content (HTML) from design, making it easier to maintain and update websites. With CSS, developers can create responsive designs that adapt to different screen sizes, ensuring a consistent user experience across devices.

CSS works by applying styles to HTML elements using rules. Each rule consists of a selector and a declaration block. The selector targets the HTML element, and the declaration block contains one or more property-value pairs that define the styling.

            
                <html> {
                    color: red;
                    }

Selector: <html> — This is the element to which styles are applied.
Rule: color — This is a property that defines the color of the text.
Value: red — This is the property value that specifies the color red.

Basic CSS Rules

  1. CSS Syntax:

    • Each CSS rule consists of a selector and a declaration block.
    • A declaration block contains one or more properties and their values, separated by semicolons (;).
    • Exemple: 
                                      selector {
                                propertie: value;
                                }

  2. Selectors:

    • Selectors determine which HTML elements the styles are applied to.
    • Basic selector types:
      • By tag: p, h1, div.
      • By class: .my-class.
      • By ID: #my-id.
      • Universal selector: * (applies to all elements).
      • Combined selectors: div.my-class, h1#my-id.

  3. Cascading:

    • When multiple rules apply to the same element, the browser uses precedence:
      • Importance (!important).
      • Specificity (selector weight).
      • Order of announcement (the last rule takes precedence).

  4. Inheritance:

    • Some CSS properties (e.g. color, font-family) are inherited by child elements from their parents.
    • Exemple: 
                                      body {
                                    font-family: Arial, sans-serif;
                                  }

  5. Box Model:

    • Each element on the page is a rectangle consisting of:
      • Content.
      • Padding.
      • Border.
      • Margin.

  6. Units of measurement:

    • Absolute: px (pixels), pt (points), cm (centimeters).
    • Relative: % (percent), em, rem (relative to font size), vh, vw (relative to browser window size).

  7. Color:

    • Colors can be specified:
      • By name: red, blue.
      • In HEX format: #ff0000.
      • In RGB/RGBA: rgb(255, 0, 0), rgba(255, 0, 0, 0.5).
      • In HSL/HSLA: hsl(0, 100%, 50%), hsla(0, 100%, 50%, 0.5).

  8. Positiong:

    • The position property determines how an element is positioned on the page:
      • static (default).
      • relative (relative to its normal position).
      • absolute (relative to the closest positioned parent).
      • fixed (relative to the browser window).
      • sticky (a combination of relative and fixed).

  9. Flexbox:

    • Used to create flexible layouts.
    • Main properties:
      • display: flex; (enables Flexbox).
      • justify-content (align to main axis).
      • align-items (alignment along the cross axis).
    • Exemple: 
                                      .container {
                                    display: flex;
                                    justify-content: center;
                                    align-items: center;
                                    }

  10. Grid:

    • Used to create complex grid layouts.
    • Main properties:
      • display: grid; (includes Grid).
      • grid-template-columns (defines columns).
      • grid-template-rows (defines rows).
      • gap (distance between elements).
    • Exemple: 
                                      .container {
                                    display: grid;
                                    grid-template-columns: 1fr 1fr 1fr;
                                    gap: 10px;
                                  }

  11. Media Queries:

    • Allows you to apply styles depending on device characteristics (screen width, orientation, etc.).
    • Exemple: 
                                      @media (max-width: 600px) {
                                    body {
                                      font-size: 14px;
                                    }
                                  }

  12. Animations and transitions:

    • Transitions: A smooth change in properties.
      •                                 button {
                                    transition: background-color 0.3s ease;
                                  }
                                  button:hover {
                                    background-color: blue;
                                  }
    • Animations (@keyframes): Create complex animations.
      •                                 @keyframes slide {
                                    from { transform: translateX(0); }
                                    to { transform: translateX(100px); }
                                  }
                                  .box {
                                    animation: slide 2s infinite;
                                  }

  13. Pseudo-classes and pseudo-elements:

    • Pseudo-classes: Allow you to style elements in a certain state.
      •                                 a:hover {
                                    color: red;
                                  }
    • Pseudo-elements: Allow you to style specific parts of an element.
      •                                 p::first-letter {
                                    font-size: 2em;
                                  }

  14. Variables (Custom Properties):

    • Allows you to create variables to reuse values.
    • Exemple: 
                                      :root {
                                    --main-color: blue;
                                  }
                                  body {
                                    background-color: var(--main-color);
                                  }

  15. Reset CSS:

    • Removes default browser styles to ensure consistency.
    • Exemple: 
                                      * {
                                    margin: 0;
                                    padding: 0;
                                    box-sizing: border-box;
                                  }
Possible options for connecting CSS

1. Inline CSS

Inline CSS is applied directly to an HTML element using the style attribute. This method is useful for quick, one-off styling but is not recommended for large-scale projects.

Exemple:
<p style="color: red; font-size: 16px;">This is a red paragraph.</p>
When to use:
For small, specific changes.
When you need to override other styles temporarily.
Pros:
Easy to implement.
High specificity (overrides external and internal styles).
Cons:
Hard to maintain.
Not reusable.
Mixes content with presentation (violates separation of concerns).

2. Internal (Embedded) CSS

Internal CSS is defined within a <style> tag in the <head> section of an HTML document. It applies styles to the entire page.

Exemple: 


                <!DOCTYPE html>
                <html>
                <head>
                    <style
                        p {
                        color: blue;
                        font-size: 18px;
                        }
                    </style>
                </head>
                <body
                    <p>This is a blue paragraph.</p>
                </body>
                </html>
When to use:
For single-page websites or small projects.
When you don’t want to create a separate CSS file.
Pros:
Keeps styles in one place.
Easier to maintain than inline CSS.
Cons:
Not reusable across multiple pages.
Increases HTML file size.

3. External CSS

External CSS is stored in a separate .css file and linked to the HTML document using the <link> tag. This is the most recommended method for larger projects.

Exemple:
1. Create a base CSS file (base.css): 

                p {
                    color: green;
                    font-size: 20px;
                    }
2. Import it into another CSS file (styles.css): 

                @import url('base.css');

                p {
                color: purple;
                }
When to use:
For modularizing CSS files.
When you want to split styles into smaller, reusable components.
Pros:
Improves organization.
Improves organization.
Can increase load time if overused.
Can increase load time if overused.
